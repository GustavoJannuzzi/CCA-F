Claude in the Browser — Your AI Workspace

Master the Claude.ai web interface: Projects for organized context, Artifacts for creative output, system prompts for precision, and document analysis for real-world productivity.

⏱ ~60 min 📚 Beginner 🎓 Everyone 🎯 3 Interactive Exercises

1. Welcome to Claude.ai

Claude.ai is the web-based interface for interacting with Claude. Available at claude.ai, it provides a rich, conversational workspace where you can chat with Claude, upload documents, generate artifacts, and manage persistent projects -- all from your browser.

Whether you are drafting a report, analyzing a contract, writing code, or brainstorming strategy, Claude.ai is designed to be your AI workspace for daily work.

The Chat Interface

When you open Claude.ai, you are greeted with a clean conversation interface. The key elements are:

  • Message input area -- at the bottom of the screen, where you type your messages. Supports rich text, file uploads, and multi-line input.
  • Model selector -- at the top, letting you choose between Claude models (Sonnet for speed, Opus for depth, Haiku for lightweight tasks).
  • Conversation panel -- the center area displaying the back-and-forth between you and Claude.
  • Projects sidebar -- on the left, showing your organized projects and conversation history.
  • Artifacts panel -- on the right, appearing when Claude generates standalone content like code, documents, or diagrams.

Creating Your First Conversation

Starting a conversation with Claude is as simple as typing in the message box and pressing Enter. There is no setup, no configuration, and no ceremony required. Just describe what you need.

Example First Message I'm preparing for a quarterly business review next week. I need to create a one-page executive summary of our Q3 results. Here are the key metrics: - Revenue: $4.2M (up 12% YoY) - Customer count: 340 (up 8%) - Churn rate: 3.1% (down from 4.2%) - NPS: 72 (up from 65) Please draft a professional executive summary with these numbers.

Claude will respond with a well-structured executive summary, formatted with markdown headings, bullet points, and clear sections. This response appears in the conversation panel and, if it is a substantial document, may also appear as an Artifact on the right side.

Understanding the Response

Claude's responses are rich and structured. Key formatting features include:

  • Markdown formatting -- Claude uses headers, bold, italics, and bullet points to organize information clearly.
  • Code blocks -- when Claude writes code, it appears in syntax-highlighted blocks with a copy button.
  • Math rendering -- Claude can render mathematical equations when discussing formulas or calculations.
  • Tables -- structured data is often presented in well-formatted tables.

The "Second Message" Technique

One of the most powerful patterns in working with Claude is the second message technique. Your first message gets a solid initial response, but the real magic happens in the follow-up.

How It Works

After Claude's initial response, send a follow-up that refines, redirects, or deepens the output. Claude retains the full context of the conversation, so your second message can be brief and specific.

  • "Make it more concise" -- cuts the response down to essentials
  • "Now rewrite it for a technical audience" -- shifts the tone and depth
  • "Add a section about risks" -- extends the content in a specific direction
  • "Actually, use a table format instead" -- restructures the entire output
  • "That's close, but our CFO prefers seeing percentages. Can you convert all absolute numbers?" -- fine-tunes details
💡

Pro tip: Your first message should give Claude the context and raw material. Your second message should direct the final shape. Think of it like sculpting: first you provide the clay, then you shape it.

2. Projects — Your Organized Workspace

Projects are Claude.ai's system for organizing your work into persistent, context-rich containers. Think of a Project as a dedicated workspace that remembers your files, your preferences, and your instructions across every conversation within it.

What Projects Are

A Project is more than a folder. It is a persistent context container that includes:

  • Uploaded files -- PDFs, spreadsheets, code files, images, and documents that Claude can reference in every conversation
  • Custom Instructions -- standing instructions that shape how Claude responds within this Project
  • Conversation history -- all past conversations organized under this Project
  • Shared context -- all conversations in the Project share the same uploaded files and instructions

Creating and Naming Projects Effectively

Good project organization is the difference between a chaotic AI experience and a streamlined workflow. Follow these naming conventions:

✅

Good Names

"Q3 Earnings Analysis", "Client Onboarding SOPs", "Product Launch - April 2025", "Legal Contract Reviews"

❌

Avoid These

"Stuff", "Work Things", "Misc", "Test" -- vague names make Projects impossible to navigate later

Adding Files and Documents

One of the most powerful features of Projects is the ability to upload files as persistent context. Once uploaded, Claude can reference these files in every conversation within the Project.

  • PDFs -- reports, contracts, research papers, manuals
  • Spreadsheets -- CSV files, data exports, financial models
  • Code files -- source code, configuration files, scripts
  • Images -- charts, diagrams, screenshots, photos
  • Text documents -- meeting notes, requirements, specifications
ⓘ

File limits: Each Project can hold multiple files, up to the context window limit. Larger files consume more of the context budget, so be selective about what you upload. Focus on the files Claude truly needs to reference.

Custom Instructions per Project

Custom Instructions are standing directives that tell Claude how to behave within a specific Project. They are applied to every conversation in the Project automatically.

Example: Custom Instructions for a Finance Project You are helping the FP&A team at Acme Corp analyze financial data. Key conventions: - All dollar amounts should be in millions (e.g., $4.2M not $4,200,000) - Always show year-over-year comparisons when discussing metrics - Flag any metric that deviates more than 10% from the prior period - Use tables for comparing multiple periods or segments - When asked to "summarize," produce a 5-bullet executive summary Our fiscal year runs April-March. Current fiscal year is FY2025.

Project Templates for Common Workflows

Once you find a Project setup that works well, you can replicate the pattern. Here are common Project templates that organizations use:

3. Artifacts — Claude's Creative Output

Artifacts are standalone pieces of content that Claude creates alongside the conversation. When Claude generates something substantial -- a document, code, a diagram, or an interactive component -- it appears in the Artifacts panel on the right side of the screen.

Think of Artifacts as Claude's "deliverables." While the conversation is the discussion, Artifacts are the tangible outputs you can use, share, and iterate on.

Types of Artifacts

Code Artifacts

Claude can produce code in virtually any programming language: Python, JavaScript, TypeScript, HTML/CSS, SQL, Rust, Go, and many more. Code Artifacts appear with syntax highlighting, a copy button, and a language label.

Document Artifacts

Formatted text documents like executive summaries, proposals, emails, and reports. These render with proper markdown formatting -- headings, bullet points, tables, and emphasis.

SVG Graphics and Diagrams

Claude can generate vector graphics directly. Logos, icons, flowcharts, and simple illustrations are rendered as SVG and displayed visually in the Artifacts panel.

Mermaid Diagrams

For more complex diagrams -- flowcharts, sequence diagrams, entity-relationship diagrams, Gantt charts -- Claude uses Mermaid syntax, which renders as interactive, professional diagrams.

React Components (Interactive HTML)

Claude can create fully interactive HTML components with embedded JavaScript. Calculators, dashboards, interactive forms, and data visualizations come to life right in the Artifacts panel.

Iterating on Artifacts

One of the most powerful aspects of Artifacts is in-place editing. Instead of regenerating an entire document, you can ask Claude to modify specific parts:

Iterating on an Artifact # After Claude generates a dashboard artifact: "Make the header background dark blue instead of gray" # Claude modifies just the header styling "Add a search filter above the data table" # Claude adds the filter component, keeping everything else "The chart should show the last 12 months, not 6" # Claude adjusts the data range in the chart

Each iteration builds on the previous version. Claude maintains the full Artifact state and makes targeted modifications, just like a human editor working on a living document.

Downloading and Sharing Artifacts

Artifacts are not trapped in Claude.ai. You can:

  • Copy to clipboard -- one click to copy the raw content (code, markdown, etc.)
  • Download as a file -- save code files, documents, or SVGs to your computer
  • Share via link -- generate a shareable link for interactive Artifacts that others can view
  • Publish -- some Artifacts can be published as standalone web pages
💡

Pro tip: When you need a complex interactive tool (a calculator, a dashboard, a form), ask Claude to create it as a React component Artifact. You will get a fully functional, interactive tool that runs right in your browser -- no deployment needed.

4. System Prompts and Custom Instructions

System prompts are instructions that set Claude's behavior before the conversation starts. They are the single most powerful tool for getting consistent, high-quality responses tailored to your specific needs.

What System Prompts Do

A system prompt tells Claude who it is, how it should behave, what constraints to follow, and how to format its responses. Think of it as a detailed job description for your AI assistant. Without a system prompt, Claude defaults to being a helpful generalist. With one, Claude becomes a specialist precisely tuned to your workflow.

The Three Types

Type Where It Lives Scope
Built-in Set by Anthropic (claude.ai defaults) Applies to all conversations by default
Project-level Project Custom Instructions Applies to every conversation in that Project
Conversation-level Your first message (or preamble) Applies to just this conversation

Patterns for Effective System Prompts

1. Persona Pattern

Define who Claude is and what expertise it brings. This shapes the vocabulary, depth, and perspective of every response.

Persona Pattern You are a senior financial analyst with 15 years of experience in SaaS metrics and enterprise valuation. You approach every question with the rigor of preparing for a board presentation. Use precise financial terminology and always consider the investor's perspective.

2. Constraint Pattern

Set clear boundaries on what Claude should and should not do. Constraints prevent hallucination, overreach, and inconsistency.

Constraint Pattern Rules: - Always cite the specific document or data source for any claim - Never fabricate statistics or make up data points - If you are unsure about something, say so explicitly - Do not provide medical, legal, or financial advice -- frame as analysis - Flag when a question falls outside your area of expertise

3. Format Pattern

Specify exactly how Claude should structure its responses. This ensures consistency across every interaction.

Format Pattern Response format: - Start with a one-sentence summary (the "bottom line up front") - Use bullet points for key findings (maximum 5) - Keep total response under 200 words - End with "Recommended next step:" followed by one action item - Use tables when comparing 3 or more items

4. Audience Pattern

Tell Claude who the reader is. This dramatically changes vocabulary, depth, and what background knowledge Claude assumes.

Audience Pattern Your audience is non-technical C-suite executives. They are brilliant strategists but do not know what an API is. Explain technical concepts using business analogies. Replace jargon with plain language. Focus on business impact, not implementation details.

Examples Across Industries

🏥

Healthcare

"You are a clinical research advisor. Always cite studies. Flag anything requiring physician consultation. Use accessible language."

⚖

Legal

"You are a legal research assistant. Identify risks and liabilities. Note jurisdiction variations. Never provide legal advice directly."

💰

Finance

"You are an FP&A analyst. Support claims with data. Use financial terminology precisely. Present findings in structured tables."

🛠

Engineering

"You are a senior software architect. Evaluate tradeoffs explicitly. Recommend proven patterns. Include code examples where helpful."

5. Document Analysis Power

One of Claude's strongest capabilities in the browser is document analysis. Upload PDFs, images, spreadsheets, and other files, then ask Claude to extract, summarize, compare, and synthesize information from them.

Uploading Documents

You can upload documents in two ways:

  • In a conversation -- click the attachment icon in the message input to upload files for a single conversation
  • In a Project -- add files to the Project's knowledge base so they persist across all conversations

Claude supports a wide range of formats: PDF, images (PNG, JPG, WebP), CSV, plain text, code files, and more. For best results, use text-based PDFs rather than scanned images when possible.

Asking Structured Questions

The key to effective document analysis is asking structured questions rather than vague ones. Compare these approaches:

Vague Question

"What does this document say?"

Gets a generic summary that may miss what you actually need.

Structured Question

"Extract all payment terms, deadlines, and penalty clauses from this contract. Present them in a table."

Gets exactly the information you need in the format you want.

Multi-Document Synthesis

Claude excels when asked to synthesize information across multiple documents. Upload several files and ask cross-cutting questions:

Multi-Document Synthesis Examples "Compare these three vendor proposals on price, timeline, and support terms. Which one offers the best overall value?" "I've uploaded our last 4 quarterly reports. Identify the top 3 trends that have been consistent across all quarters." "Here are resumes from 5 candidates. Create a comparison matrix of their skills, experience levels, and cultural fit indicators."

Data Extraction Patterns

Claude is exceptionally good at extracting structured data from unstructured documents:

  • "Extract all dates and amounts from this invoice" -- turns messy invoices into clean data
  • "List every person mentioned in this report with their title and role" -- builds a stakeholder map
  • "Pull out all action items from these meeting notes" -- converts notes into a task list
  • "Identify all regulatory requirements mentioned in this compliance document" -- creates a checklist

Vision Capabilities

Claude can analyze images, charts, screenshots, and photos. This makes it valuable for:

  • Chart analysis -- upload a chart and ask Claude to describe trends, identify anomalies, or extract data points
  • Screenshot review -- share a UI screenshot and ask for feedback on design, usability, or content
  • Handwritten notes -- upload photos of whiteboard sessions or handwritten notes for transcription
  • Infographic interpretation -- ask Claude to describe and explain complex visual information
⚠

Important: While Claude's vision is powerful, it is not perfect for fine-grained text extraction from complex images. For best results with PDFs, use text-based (not scanned) PDFs. For scanned documents, consider using OCR first, then uploading the text.

6. Power Patterns for Daily Work

Beyond basic prompting, experienced Claude users employ a set of power patterns that consistently produce higher-quality results. These patterns work in any domain and scale from simple questions to complex multi-step workflows.

Chain-of-Thought

Asking Claude to "think step by step" before answering forces it to reason through complex problems rather than jumping to a conclusion. This is especially valuable for math, logic, analysis, and any task where the reasoning matters as much as the answer.

Chain-of-Thought Pattern Before answering, think step by step: 1. What are the key factors to consider? 2. What are the tradeoffs between each option? 3. What does the data actually show? Then provide your recommendation with supporting reasoning.

Role-Play

Ask Claude to adopt a specific perspective to stress-test your ideas and find blind spots.

  • "Play devil's advocate on my proposal" -- finds weaknesses before your critics do
  • "Respond as if you were a skeptical investor hearing this pitch" -- tests your business case
  • "Review this as if you were a QA engineer looking for edge cases" -- finds bugs and gaps
  • "What would a regulator's concerns be with this approach?" -- identifies compliance risks

Multi-Document Synthesis

When you upload 5 or more documents, Claude can answer cross-cutting questions that would take hours to research manually:

Cross-Cutting Analysis I've uploaded our last 6 monthly sales reports. Please: 1. Identify any trends that appear across 3+ months 2. Flag any anomalies (metrics that deviate >15% from average) 3. Correlate marketing spend with lead generation numbers 4. Summarize in a table: Month | Revenue | Leads | Cost per Lead | Trend

Template Generation

Create reusable prompt templates that you can fill in each time. This is especially powerful for recurring tasks like weekly reports, client emails, or code reviews.

Reusable Template # Weekly Status Report Template Generate a weekly status report using this structure: Team: [TEAM_NAME] Week ending: [DATE] 1. ACCOMPLISHMENTS (top 3 completed items) 2. IN PROGRESS (items currently being worked on, with % complete) 3. BLOCKERS (anything preventing progress) 4. NEXT WEEK PRIORITIES (top 3 planned items) 5. METRICS: [METRIC_1], [METRIC_2], [METRIC_3] Keep each section to 2-3 bullet points. Total length: under 1 page.

Conversation Branching

Claude.ai allows you to start a new response from any previous message in the conversation. This is called branching, and it lets you explore alternatives without losing your original thread:

  • Explore different approaches -- branch from a decision point to see two different solutions side by side
  • Undo and redo -- if Claude goes in the wrong direction, branch from an earlier message and redirect
  • A/B test prompts -- branch from the same point with two different instructions to compare outputs
💡

Power user move: Combine multiple patterns in one prompt. Start with a persona ("You are a senior data analyst"), add constraints ("Use only the uploaded data, never fabricate numbers"), specify the format ("Present as a 5-bullet executive summary"), and trigger chain-of-thought ("Show your reasoning before the final summary"). This layered approach consistently produces the best results.
