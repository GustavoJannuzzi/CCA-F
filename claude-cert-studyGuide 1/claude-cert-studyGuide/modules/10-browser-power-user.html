Browser Power User

Go beyond basic chat. Master Projects for persistent context, create stunning artifacts, build industry personas, run advanced research workflows, and create prompt templates your whole team can use.

⏱ ~75 min 📚 Intermediate 🎯 3 Interactive Exercises 👥 Consultants, Business Users, Leaders

1. Advanced Projects

Claude Projects are your secret weapon for complex work. While a single conversation loses context once you close it, a Project maintains persistent knowledge across every conversation you start within it. Think of a Project as a dedicated workspace with built-in memory.

Multi-File Context: Uploading 10+ Documents

The real power of Projects emerges when you upload multiple documents. Claude can hold the contents of all uploaded files in its context simultaneously, enabling cross-document reasoning that would be impossible in a single conversation.

Example: Quarterly Business Review Prep

Upload all of these into a single Project:

  • Q3 financial report (PDF)
  • Customer satisfaction survey results (CSV)
  • Competitor analysis deck (PDF)
  • Product roadmap (PDF)
  • Sales pipeline export (CSV)
  • Engineering velocity metrics (PDF)
  • Board meeting minutes from last quarter (DOCX)
  • Market research report (PDF)
  • Customer churn analysis (XLSX)
  • Employee engagement survey (PDF)

Now ask: "Based on all uploaded documents, what are the three biggest risks to hitting our Q4 targets, and what evidence from these documents supports each risk?"

Claude will synthesize across all 10 documents, citing specific data points from each.

Version Control: Iterating Across Conversations

Within a Project, every conversation has access to the same uploaded documents and Project instructions. This means you can:

  • Start fresh conversations without re-explaining context -- Claude already knows your project
  • Iterate on documents across multiple sessions -- start a strategy doc in one conversation, refine it in the next
  • Compare outputs -- ask the same question in different conversations with different framing to get diverse perspectives
  • Maintain a clean history -- each conversation stays focused on one topic while sharing the same foundational knowledge

Project Templates: Reusable Setups

For recurring engagements, create template Projects that you can duplicate and customize. This saves significant setup time.

📋

Client Onboarding

Upload standard intake questionnaire, proposal template, and industry benchmarks. Customize per client.

📊

Due Diligence

Pre-load financial analysis frameworks, regulatory checklists, and valuation model templates.

📝

Content Strategy

Include brand voice guide, editorial calendar template, competitor content audit, and style guide.

🎓

Training Development

Load learning objectives framework, assessment rubrics, and instructional design models.

Sharing and Collaboration

Projects can be shared with team members within the same Claude organization. When you share a Project, collaborators gain access to the same uploaded documents and Project instructions, ensuring consistent context. Key sharing considerations:

  • Shared context, separate conversations -- each team member has their own conversation history
  • Permission levels -- control who can upload new documents or modify Project instructions
  • Organization-wide templates -- publish Project templates for your entire team to use
  • Sensitive data awareness -- remember that uploaded documents are accessible to all Project members
💡

Pro tip: Write detailed Project instructions (the "system prompt" for the Project) that explain what the uploaded documents are, how they relate to each other, and what kinds of questions the Project is designed to answer. This dramatically improves Claude's responses.

2. Artifact Mastery

Artifacts are Claude's ability to create substantial, standalone content that appears in a dedicated panel beside the conversation. Unlike inline code snippets or short text responses, artifacts are complete, functional pieces of work you can preview, iterate on, and download.

Code Artifacts: Full Applications

Claude can generate complete, runnable applications -- not just code snippets. These render live in the artifact panel so you can interact with them immediately.

  • Interactive HTML games -- Snake, Tetris, memory cards, quiz apps
  • Data visualization dashboards -- charts, graphs, interactive filters
  • Utility tools -- calculators, converters, form builders
  • Python scripts -- data processing, file manipulation, API clients
  • Full React components -- stateful UI with animations and event handling
🛈

Key insight: HTML artifacts run in a sandboxed iframe with full CSS and JavaScript support. This means you can create interactive applications with state management, animations, and user input -- all rendered live.

Mermaid Diagrams

Claude excels at generating Mermaid diagrams -- a text-based diagramming language that renders as visual diagrams. Supported types include:

  • Flowcharts -- business processes, decision trees, workflows
  • Sequence diagrams -- API call flows, system interactions, protocol exchanges
  • ER diagrams -- database schemas, data models, entity relationships
  • Gantt charts -- project timelines, sprint planning, resource allocation
  • State diagrams -- state machines, lifecycle models
  • Class diagrams -- object models, inheritance hierarchies
Example Prompt "Create a Mermaid sequence diagram showing the OAuth 2.0 authorization code flow between a User, Client App, Authorization Server, and Resource Server. Include token refresh."

SVG Graphics

For data visualizations, icons, and infographics, Claude can generate SVG graphics that render at any resolution. This is particularly useful for:

  • Custom data charts -- bar charts, pie charts, scatter plots with your specific data
  • Infographics -- process flows, comparison graphics, timeline visuals
  • Icons and illustrations -- simple graphics for presentations or documentation
  • Architectural diagrams -- system architecture with custom styling

React Components

Claude can create interactive React components that run directly in the artifact panel. These support state management, event handling, and CSS animations -- making them ideal for prototyping UI concepts, building interactive demos, or creating tools.

Iterating Effectively

One of the most powerful aspects of artifacts is in-place editing. Rather than regenerating the entire artifact, Claude can make targeted changes:

  • "Move the legend to the right side of the chart"
  • "Change the color scheme to use blues instead of reds"
  • "Add a tooltip that shows the exact value when hovering over a bar"
  • "Make the animation slower and add an easing curve"

Each iteration preserves the parts of the artifact you liked while making targeted improvements. You can also refer to specific parts: "In the second section, change the heading from 'Overview' to 'Executive Summary' and make the bullet points more concise."

The Artifact Versioning Model

Every time Claude modifies an artifact, it creates a new version. You can navigate between versions using the version controls in the artifact panel. This means you can experiment freely -- if an iteration goes in the wrong direction, simply revert to a previous version and try a different approach.

⚠

Watch out: Artifacts have a maximum size. Very large applications (thousands of lines) may need to be split across multiple artifacts. For complex projects, ask Claude to create a modular structure with separate artifacts for different components.

3. Building Industry Personas

A generic Claude response is helpful. A Claude response shaped by a well-crafted industry persona is transformative. The persona pattern is arguably the single most impactful technique for business users.

The Persona Pattern

The core structure is simple but powerful:

Persona Pattern "You are a [role] with [expertise] who [specializes in]. Always [behavioral rule 1]. [Behavioral rule 2]. [Output format preference]. [Constraint]."

The key components are:

  • Role -- the professional identity (e.g., "senior financial analyst")
  • Expertise -- specific domain knowledge (e.g., "specializing in equity valuation and DCF modeling")
  • Behavioral rules -- how the persona should communicate (e.g., "always use precise financial terminology")
  • Output format -- structural preferences (e.g., "present analysis in structured sections with clear metrics")
  • Constraints -- guardrails (e.g., "flag any assumptions explicitly")

Industry-Specific Personas

Finance

Finance Persona You are a senior financial analyst at a Fortune 500 investment firm. You specialize in equity valuation, DCF modeling, and earnings analysis. Always use precise financial terminology. Support all claims with quantitative reasoning. Present analysis in structured sections with clear metrics. Flag any assumptions explicitly.

Healthcare

Healthcare Persona You are a clinical research advisor specializing in evidence-based medicine and healthcare operations. Always cite relevant clinical studies and guidelines. Use accessible language while maintaining medical accuracy. Flag any information that requires physician consultation. Focus on patient outcomes and quality metrics.

Legal

Legal Persona You are a legal research associate specializing in M&A due diligence and contract analysis. Identify potential risks, liabilities, and regulatory considerations. Always note jurisdiction-specific variations. Never provide specific legal advice -- frame as research findings and flag items for attorney review. Use structured analysis with clear issue spotting.

Retail

Retail Persona You are a retail strategy consultant specializing in omnichannel customer experience, conversion optimization, and competitive positioning. Focus on data-driven recommendations with clear ROI projections. Use retail-specific KPIs (conversion rate, AOV, LTV, CAC, NPS). Reference industry benchmarks from NRF, Forrester, and McKinsey.

Testing Personas

Before rolling out a persona to your team, test it rigorously:

  1. Standard question test -- ask the same question with and without the persona. The difference should be dramatic.
  2. Edge case test -- ask questions outside the persona's domain. It should stay in character while acknowledging limitations.
  3. Consistency test -- ask similar questions multiple times. The tone, terminology, and depth should be consistent.
  4. Guardrail test -- try to get the persona to violate its constraints. Verify it maintains its behavioral rules.

Sharing Persona Prompts Across Teams

Once validated, distribute your persona prompts through:

  • Project instructions -- embed the persona in a shared Project so every conversation uses it
  • Team prompt library -- maintain a document with approved personas, organized by function and use case
  • Onboarding materials -- include persona prompts in new team member training
  • Regular reviews -- update personas quarterly as industry terminology and priorities evolve

4. Research & Analysis Workflows

Claude transforms research from a tedious, multi-day process into a structured, efficient workflow. The key is knowing which techniques to apply and in what order.

Multi-Document Synthesis

The most powerful research technique is cross-document synthesis: uploading multiple related documents and asking Claude to find patterns, contradictions, and insights across all of them.

Example: Analyst Report Comparison

Upload 5 analyst reports on the same company, then ask:

"Compare the revenue forecasts across all 5 analyst reports. Create a table showing each analyst's projection for FY25 and FY26. Identify where they agree, where they disagree, and what assumptions drive the differences. Cite the specific page and paragraph for each claim."

Competitive Analysis Framework

Claude can apply established strategic frameworks systematically. Rather than generating generic analyses, point Claude at your uploaded competitive intelligence and ask for specific frameworks:

Framework Best For Example Prompt
SWOT Analysis Quick strategic overview "Based on our uploaded market data, create a SWOT analysis for [Company]"
Porter's Five Forces Industry structure analysis "Analyze the competitive intensity of [industry] using Porter's Five Forces, citing our uploaded reports"
Blue Ocean Strategy Market differentiation "Using the uploaded competitor data, create a strategy canvas and identify blue ocean opportunities"
Value Chain Analysis Operational advantage "Map the value chain for [Company] and identify where they have cost or differentiation advantages"

Citation Tracking

For any research work, citation tracking is critical. Without it, you cannot verify Claude's claims or present findings to stakeholders with confidence. Always include citation requirements in your prompts:

Citation Prompt Pattern "For every factual claim in your analysis, cite the source document by name and the specific section or page number. If a claim is based on your own reasoning rather than the uploaded documents, mark it as [Inference]. If you are uncertain about a data point, mark it as [Verify]."

Executive Briefings

One of the highest-value workflows is transforming raw data into a structured executive summary. The key is specifying the exact format:

Executive Briefing Prompt "Based on all uploaded documents, create a one-page executive briefing with the following structure: 1. SITUATION (2-3 sentences): What is happening 2. KEY FINDINGS (3-5 bullet points): The most important insights 3. RISK FACTORS (2-3 bullet points): What could go wrong 4. RECOMMENDATION (1-2 sentences): What we should do 5. NEXT STEPS (numbered list): Specific actions with owners Keep language crisp and direct. No jargon. Every claim must reference a specific uploaded document."

Data Storytelling

Raw numbers rarely persuade. Claude can transform spreadsheet data into compelling narratives that highlight trends, anomalies, and actionable insights:

  • Trend narratives -- "Revenue grew 12% YoY, but this masks a concerning shift: enterprise deals grew 28% while SMB declined 8%, suggesting increasing customer concentration risk"
  • Benchmark comparisons -- "Our 38-day DSO compares favorably to the industry median of 45 days, but top-quartile performers achieve 28 days"
  • Anomaly detection -- "March showed an unusual 40% spike in returns. Cross-referencing with the customer feedback data, this correlates with the firmware update released on March 3"
💡

Power move: Upload a spreadsheet and ask Claude to "tell me the three most surprising things in this data and explain why they matter for our business." Claude often surfaces insights that would take hours to find manually.

5. Prompt Templates for Teams

A prompt template is a reusable, parameterized prompt designed for a specific task. Instead of every team member writing ad-hoc prompts (with wildly varying quality), templates ensure consistent, high-quality outputs across your organization.

Template Anatomy

Every effective template has four components:

Context

Background information that sets the stage. This includes the persona, the domain, and any relevant constraints. Example: "You are a senior proposal writer for a technology consulting firm..."

Specific Task

The exact action you want Claude to perform, with clear parameters. Example: "Write a response to the following RFP section: [PASTE RFP SECTION]"

Format

How the output should be structured. Example: "Structure the response with: (1) Understanding of the requirement, (2) Our approach, (3) Relevant experience, (4) Timeline and deliverables."

Constraints

Guardrails and quality standards. Example: "Keep the total response under 500 words. Do not make claims about capabilities we have not demonstrated. Use active voice."

Templates by Function

Sales

RFP Response Generator

You are a senior proposal writer for [COMPANY]. Our key differentiators are: [LIST 3-5 DIFFERENTIATORS]. Write a compelling response to this RFP section: [PASTE RFP SECTION] Structure: (1) Understanding, (2) Approach, (3) Experience, (4) Deliverables. Keep under 500 words. Use active voice. Highlight our differentiators naturally -- never list them as bullet points. Include a brief relevant case study.
Marketing

Social Media Post Series

You are a social media strategist for [BRAND] in the [INDUSTRY] space. Our tone is [TONE: professional yet approachable / bold and provocative / warm and educational]. Create a 5-post series for [PLATFORM] about [TOPIC]. For each post: - Hook (first line that stops the scroll) - Body (2-3 sentences, value-driven) - CTA (clear next step) - 3-5 relevant hashtags - Best posting time recommendation Vary the format: 1 story, 1 statistic, 1 question, 1 tip, 1 case study.
Finance

Earnings Call Summary

You are a financial analyst summarizing an earnings call. Summarize the following transcript: [PASTE or UPLOAD] Structure: 1. KEY METRICS: Revenue, EPS, margins vs. consensus 2. GUIDANCE: Forward-looking statements and targets 3. STRATEGIC HIGHLIGHTS: Major announcements 4. RISK FACTORS: Concerns raised by management or analysts 5. ANALYST SENTIMENT: Tone of Q&A questions 6. ACTIONABLE TAKEAWAYS: 3 things an investor should know Use precise financial language. Include specific numbers. Flag any guidance changes vs. prior quarter.
HR

Job Description Generator

You are an HR specialist writing inclusive, compelling job descriptions for [COMPANY]. Create a job description for: [ROLE TITLE] Department: [DEPARTMENT] Level: [JUNIOR / MID / SENIOR / LEAD] Location: [LOCATION / REMOTE / HYBRID] Include: - Engaging opening paragraph (why this role matters) - 5-7 key responsibilities (start with impact verbs) - Required qualifications (be specific, avoid wish lists) - Preferred qualifications (clearly marked as preferred) - What we offer (compensation philosophy, benefits, culture) Avoid gendered language. Do not require unnecessary credentials. Focus on skills over years of experience.
Engineering

Technical Spec Writer

You are a senior software engineer writing a technical specification for peer review. Feature: [FEATURE DESCRIPTION] Context: [RELEVANT SYSTEM CONTEXT] Write a technical spec with: 1. OVERVIEW: Problem statement and proposed solution 2. GOALS / NON-GOALS: Explicit scope boundaries 3. DESIGN: Architecture, data model, API contracts 4. ALTERNATIVES CONSIDERED: Other approaches and why not 5. ROLLOUT PLAN: Feature flags, migration, monitoring 6. OPEN QUESTIONS: Decisions needing team input Be specific enough that another engineer could implement from this spec. Include diagrams where helpful (describe them as Mermaid code). Flag any security considerations.

Quality Testing Before Distribution

Before sharing a template with your team, run it through a quality validation process:

  1. Diverse inputs test -- run the template with 5+ different inputs that span the expected range
  2. Edge case test -- try unusual or minimal inputs. Does the template still produce useful output?
  3. Consistency test -- run the same input 3 times. Are the outputs structurally consistent?
  4. Stakeholder review -- have someone from the target audience evaluate the outputs for accuracy and usefulness
  5. Version the template -- track changes and improvements over time with clear version numbers
⚠

Common mistake: Teams often create templates that are too vague ("Write a good email about [TOPIC]") or too rigid ("Write exactly 3 paragraphs with exactly 50 words each"). The best templates give Claude clear structure while leaving room for judgment on details like length, tone nuances, and level of technical depth.
